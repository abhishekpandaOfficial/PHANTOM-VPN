First-time onboarding guide

Use PHANTOM the right way.

This page explains the real onboarding flow for a new PHANTOM VPN user. It also clarifies the current TOTP behavior so users do not end up on the admin dashboard by mistake.

End-user summary: visit /vpn, create your account, scan the WireGuard QR or download the config, open your /profile link, connect the VPN, then keep using the same profile if you upgrade.

Important clarification

TOTP is admin-only Authenticator codes are used only on the PHANTOM admin and monitoring dashboards.
User QR is a VPN QR The QR shown to a normal user after signup is a WireGuard configuration QR, not a TOTP QR.
User access is profile-based Users open their personal PHANTOM profile page to see connection status, remaining trial time, and payment actions.

First-time user steps

Visit the VPN page Open https://phantom.origixcloud.com/vpn and review the PHANTOM VPN overview.
Create your account Enter your name, email address, and an optional device note. Submit the form to create a dedicated VPN profile.
Save your profile details After signup you will see a success panel with your WireGuard QR, config download, and personal profile link.
Import the VPN in WireGuard Open WireGuard on phone or desktop. Choose the QR import option or import the downloaded .conf file.
Open your PHANTOM profile page Visit your generated profile link to confirm the account status, free time remaining, and later payment state.
Connect the VPN Turn the tunnel on inside WireGuard. Your PHANTOM profile page should update with active connection details.
Continue after trial If the free hour is finished, pay with Razorpay or submit a manual payment reference. Approval keeps the same profile active.

Connection checklist

Before connecting

Make sure the WireGuard app is installed and the PHANTOM VPN profile has been imported successfully.

While connecting

Switch the tunnel on in WireGuard, then refresh your PHANTOM profile page if the status does not update immediately.

If QR is missing later

The profile page can re-show the QR only if the same browser still has the original config saved locally. Keep the downloaded file as backup.

If payment is required

Use the payment section on your profile page. Manual payment references stay visible for admin review until approved.

Which page should a new user open first?

The correct starting point is /vpn.

Which page should a returning user open?

The returning user should open their personal /profile?user=... link.

Which page needs the authenticator code?

Only /admin and /dashboard require the admin authenticator code.
